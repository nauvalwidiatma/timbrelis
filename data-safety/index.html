Current App Behavior

  • Microphone permission is used for realtime audio visualization and analysis.
  • Audio is processed locally on the device.
  • Audio can be saved as local WAV files only when the user starts recording.
  • Finished recordings can be exported to local device media storage when the user chooses.
  • Audio is not uploaded.
  • Audio is not shared with third parties.
  • No account is required.
  • No ads SDK is included in the current version.
  • No analytics SDK is included in the current version.
  • No crash reporting SDK is included in the current version.

Google Play Data Safety Draft

For the current no-network, no-ads, no-analytics build: data collected: no; data shared: no; microphone access: used for app functionality with realtime local processing, user-initiated local recording, and user-initiated local export.

Future Updates

If Google AdMob, Google Analytics for Firebase, Firebase Crashlytics, cloud sync, account login, subscriptions, or any network data processing is added, this page and the Google Play Data safety form must be updated before release.
